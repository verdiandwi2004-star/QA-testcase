# QA Test Case Documentation – Tokopedia Website
This repository contains structured functional testing documentation based on UI/UX analysis of the Tokopedia website.
The purpose of this study case is to simulate real-world QA workflow including test case development, bug identification, and severity classification.

## Scope of Testing
- Product Search
- Filter & Sorting
- Cart Management
- Checkout Process
- Payment Flow
- Order Tracking
- Profile Management
- Seller Interaction

## Testing Approach
- Blackbox Testing
- Functional Testing
- UI/UX-based validation
- Severity & Priority classification
- Positive & Negative test scenarios

## Documentation Structure
Each test case includes:
- Test Case ID
- Module / Feature
- Test Description
- Preconditions
- Test Steps
- Expected Result
- Actual Result
- Status (Pass/Fail)
- Severity
- Priority
- Evidence
